2. How the Engine Works (The Deepest Technical Knowledge)
If the interviewer asks, "Walk me through your methodology," this is your answer. The project is broken into 4 core Python models.
A. The Demand Model (models/demand.py)
What it does: Predicts how many EVs will be on the road in each state by 2030, and how much electricity (MWh) they will need.
How it works (The Math): It doesn't just guess. It takes 5 years of historical EV registration data (2020-2024) and fits an exponential growth curve using Ordinary Least Squares (OLS) regression. Specifically, it plots ln(EV registrations) = a + b * year to find the Compound Annual Growth Rate (CAGR) for each state. It caps extreme growth at 120% to stay realistic.
Smart Feature: It includes Scenario Analysis (Low, Base, High adoption rates) multiplying demand by 0.65x or 1.45x.
B. The Financial Model (models/financial.py)
What it does: Calculates if building a 10 Megawatt (MW) Solar + EV charging hub is actually profitable.
How it works (The Finance):
It calculates NPV (Net Present Value): How much profit the project makes over 20 years in today's money, assuming a 10% discount rate.
It calculates IRR (Internal Rate of Return): The annualized percentage return of the project. Technical flex: The code doesn't use a basic Excel formula; it uses a Bisection Search Algorithm to find the exact IRR by finding the rate where NPV equals zero.
The "Policy" Layer: It runs the financials twice. Once for the base case, and once factoring in government subsidies (FAME-II scheme, accelerated depreciation, state GST waivers).
The "Risk" Layer (Monte Carlo): This is your biggest flex. Instead of assuming one fixed price, it runs a Monte Carlo Simulation (1,000 iterations). For each state, it randomizes the Grid Tariff (±15%), Solar sunlight (±8%), and Construction Costs (±10%). It outputs P10 (worst-case), P50 (expected), and P90 (best-case) returns.
C. The Grid Stress Model (models/grid.py)
What it does: Ensures the local power grid won't blow up when thousands of EVs plug in.
How it works: It calculates the peak GW (Gigawatt) load of the new EVs and compares it to the state's existing peak demand. If EV demand adds >10% to the grid load, the state is flagged as "Critical". It combines this with the financial health of the state's power company (DISCOM) to create a "Grid Readiness Score."
