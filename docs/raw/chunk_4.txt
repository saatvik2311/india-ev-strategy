🔹 1. The core idea: What is IRR really?
IRR (Internal Rate of Return) is:
👉 The exact discount rate where your project “breaks even” in today’s money
Not accounting profit.
Not revenue.
👉 Value today = 0
🔹 2. What is NPV (quick refresher)
= cash flow at year t
= discount rate
🔸 Interpretation:
NPV > 0 → project makes money
NPV < 0 → project loses money
NPV = 0 → perfect break-even
🔹 3. Why do we set NPV = 0?
This is your key confusion—so let’s make it intuitive.
🧠 Think like an investor
You’re asking:
“What return rate am I actually earning from this project?”
Try different discount rates:
👉 At 15%, something special happens:
Future cash flows exactly cancel initial investment
No extra gain, no loss
🔥 That rate = IRR
👉 “This project behaves like a 15% return investment”
🔹 4. Why can’t we solve it directly?
Because:
👉 r is inside multiple denominators with powers
This becomes a non-linear equation
👉 No simple algebra solution
🔹 5. So we use Bisection (Binary Search)
You already know binary search—so here’s the mapping:
🔸 Goal:
Find r such that:
🔸 Setup:
lo = 0.001   # 0.1%
hi = 5.0     # 500%
🔸 Algorithm logic:
Take midpoint:
mid = (lo + hi) / 2
Compute:
npv = NPV(mid)
🔸 Decision rule:
If NPV(mid) > 0
👉 rate too low → increase it
👉 lo = mid
If NPV(mid) < 0
👉 rate too high → decrease it
👉 hi = mid
🔁 Repeat ~80 times
Each time:
👉 You cut the search space in half
🔹 6. Why does this work? (intuition)
Because:
👉 NPV decreases as discount rate increases
🔸 Visual idea:

        IRR
Left side → positive NPV
Right side → negative NPV
Crossing point → NPV = 0
🔹 7. Why “zero” specifically?
Now the real answer:
🔥 Because IRR is a RATE, not a value
You are not asking:
❌ “How much profit?”
✅ “What return rate does this project generate?”
Setting NPV = 0 converts:
👉 cash flows → a single percentage
🔸 Analogy
You invest ₹100:
Get ₹110 → 10% return
Get ₹150 → 50% return
For complex projects:
👉 Cash flows are spread over 20 years
So you ask:
“What constant interest rate gives me the same outcome?”
That’s IRR.
